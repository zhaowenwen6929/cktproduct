# 生成过程支持取消 PRD

## 一、背景
当前对话侧边栏已支持图片与视频生成，但用户在生成链路中需要明确知道：任务是否已真正进入生成、当前是否还能取消、取消后是否返还积分，以及取消后如何继续操作。若取消规则按阶段不透明，容易引发“为什么这次返积分、那次不返”“为什么点击发送后不能再发起新任务”“取消后下一步该做什么”这类理解成本和体验落差。

基于当前实际实现，生成任务的取消并不是单一动作，而是与任务阶段强相关：在需求提交和解析阶段可直接取消；在排队阶段取消需要确认并返还积分；在正式生成阶段取消需要确认且不返还积分。与此同时，生成过程中还存在慢提醒、完成通知、成功/失败反馈等联动状态，因此需要将“生成过程支持取消”按当前实现重写为结构化 PRD，便于开发、测试、产品使用同一口径对齐。

## 二、页面定位
- 页面形态：对话侧边栏生成链路 / AI功能操作面板
- 主类型：AI功能与提示词类
- 子类：多步骤流程类 / 消息通知类
- 置信度：高

## 三、功能清单
| 模块 | 功能点 | 说明 |
|---|---|---|
| 发送入口 | 发送/停止按钮切换 | 无任务时发送，有任务时停止 |
| 状态流转 | submitting / analyzing / queued / generating | 生成任务按阶段流转 |
| 取消策略 | 分阶段取消 | 不同阶段取消方式、确认方式、积分规则不同 |
| 积分处理 | queued 扣减 / 指定阶段返还 | 排队时扣积分，取消或异常时按规则返还 |
| 取消反馈 | 写入取消结果消息 | 取消后输出明确反馈与下一步引导 |
| 慢提醒联动 | 取消后结束慢提醒 | 生成中慢提醒与取消状态联动关闭 |
| 完成通知联动 | 取消后不再触发完成提醒 | 取消后终止完成结果回写 |

## 四、菜单权限
| 权限项 | 说明 | 控制方式 |
|---|---|---|
| 发送任务 | 允许发起图片/视频生成 | 输入有内容且无进行中任务时可用 |
| 停止任务 | 允许取消当前任务 | 有进行中任务时发送按钮切为停止 |
| 确认取消 | 允许在指定阶段确认取消 | queued / generating 阶段通过弹窗确认 |
| 接收通知 | 允许开启完成提醒 | 浏览器通知权限允许时可开启 |

## 五、模块需求

### 5.1 发送与停止入口 [1]
- 输入为空时，不允许发送，发送按钮置灰。
- 当前不存在进行中任务时，按钮语义为“发送”，图标为发送箭头。
- 当前存在进行中任务时，按钮语义切换为“停止”，图标切换为停止方块。
- 任务进行中时，用户不可再次发起新任务；点击按钮只触发取消链路。

### 5.2 状态流转与取消策略 [2]
生成任务的前端状态流转按当前实现固定为：
1. `submitting`
2. `analyzing`
3. `queued`
4. `generating`
5. `completed / failed / cancelled`

各阶段取消策略如下：
- `submitting`：可直接取消，无确认弹窗，本阶段未扣积分。
- `analyzing`：可直接取消，无确认弹窗，本阶段未扣积分。
- `queued`：点击停止后弹确认弹窗；确认取消后返还已扣积分。
- `generating`：点击停止后弹确认弹窗；确认停止后终止任务且不返还积分。

### 5.3 积分规则 [3]
- 积分不是在点击发送时扣减，而是在任务进入 `queued` 阶段时扣减。
- 在 `submitting` 或 `analyzing` 取消：未扣积分，无返还动作。
- 在 `queued` 取消：返还全部已扣积分。
- 在 `generating` 取消：不返还积分。
- 任务执行异常且已扣积分：返还全部已扣积分。

### 5.4 取消确认与结果反馈 [4]
#### 5.4.1 queued 阶段确认弹窗
- 标题：`确认取消排队中的任务？`
- 说明：当前任务仍在等待生成，建议继续等待结果返回。若现在取消，系统将返还积分。
- 主按钮：`确认取消并返还积分`
- 次按钮：`继续等待`

#### 5.4.2 generating 阶段确认弹窗
- 图片标题：`确认停止当前图片生成？`
- 视频标题：`确认停止当前视频生成？`
- 说明：当前任务已进入正式生成阶段，建议耐心等待结果返回。若现在取消，将停止任务且不返还积分。
- 主按钮：`确认停止且不返还积分`
- 次按钮：`继续等待`

#### 5.4.3 取消成功后的消息反馈
取消成功后，系统需写入一条助手消息：
- `submitting/analyzing`：`已取消当前任务，本阶段尚未扣除积分。
您可以编辑输入信息后再次发送，重新生成。`
- `queued`：`已取消当前任务，X 积分已返还。
您可以编辑输入信息后再次发送，重新生成。`
- `generating`：`已停止当前图片/视频生成任务，积分不返还。
您可以编辑输入信息后再次发送，重新生成。`

### 5.5 慢提醒与完成通知联动
- 慢提醒仅在任务进入 `generating` 且超过 5 秒未返回预览结果时展示。
- 任务取消后，慢提醒文案与相关展示状态需同步结束。
- 任务取消后，不再继续写入正常完成消息，也不再触发完成通知。
- 当异步结果在取消后晚到时，需通过取消标记阻断后续正常完成流程。

## 六、交互说明
| 场景 | 交互 | 结果 |
|---|---|---|
| 发送任务 | 输入有效内容后点击发送 | 创建任务并进入状态流转 |
| 停止任务 | 任务进行中点击停止 | 根据阶段执行直接取消或弹窗确认 |
| queued 取消确认 | 点击确认取消并返还积分 | 取消任务并返还积分 |
| generating 取消确认 | 点击确认停止且不返还积分 | 停止任务且不返还积分 |
| 取消后继续等待 | 点击“继续等待” | 关闭弹窗，任务继续 |
| 取消完成 | 系统写入助手消息 | 明确当前取消结果与后续引导 |

## 七、异常与边界
| 异常/边界 | 处理方式 |
|---|---|
| 输入为空 | 不允许发送，发送按钮置灰 |
| 积分不足 | 不创建任务，不清空输入框，输出积分不足提示 |
| 任务进行中重复点击发送 | 按钮语义切为停止，不允许发起新任务 |
| 任务取消后异步结果晚到 | 通过取消标记阻断成功消息和完成逻辑 |
| 浏览器通知权限为 denied | 不允许开启完成提醒，显示权限受限文案 |
| 生成中预览位数量固定 | 当前固定 4 宫格展示，占位不随请求数量动态变化 |

## 九、验收标准
1. 任务进行中时，发送按钮必须切换为停止按钮。
2. `submitting/analyzing` 阶段点击停止后可直接取消，无确认弹窗。
3. `queued` 阶段点击停止后必须弹确认弹窗，确认后返还积分。
4. `generating` 阶段点击停止后必须弹确认弹窗，确认后不返还积分。
5. 积分扣减时机必须发生在进入 `queued` 阶段，而不是点击发送时。
6. 取消成功后必须写入对应阶段的助手反馈消息，并包含“编辑输入信息后再次发送，重新生成”的引导。
7. 任务取消后，慢提醒和完成通知链路必须停止，不允许继续写入正常完成结果。
8. 任务取消后即使异步结果晚到，也不能继续走成功完成逻辑。

## 十、待确认项

- queued 阶段的积分返还目前是前端状态回补，是否需要以后端账务结果为准仍待确认。
- 是否需要为取消链路补充独立埋点与监控日志仍待确认。
