# 导出作品弹窗 PRD

## 一、背景
当前导出能力主要面向画布内容的交付与下载场景，但现有流程只支持将选中内容合并为单张图片导出，无法满足多个素材分别导出的诉求。与此同时，画布已经支持视频内容，但导出入口在视觉上仍主要呈现 PNG、JPG 等图片格式，用户点击导出后容易误以为视频不能导出，导致导出能力表达不完整、预期不明确。

这一问题会直接影响素材交付效率：有些场景需要将多个元素分别下载用于后续加工或复用，而不是强制合成一张图；同时，视频内容已经进入画布能力范围，如果导出入口不能同步体现视频导出方式，用户会对可导出的结果产生误判。

因此，需要补充「导出方式」能力，让用户可以明确区分直接导出与合并导出，并在导出入口中正确识别当前内容可导出的类型与结果。

## 二、页面定位
- 页面形态：导出弹窗 / 画布工具类配置弹层
- 主类型：后台表单编辑类
- 子类：上传导入类 / 详情展示类
- 置信度：高

## 三、功能清单
| 模块 | 功能点 | 说明 |
|---|---|---|
| 导出入口 | 从顶部栏触发导出 | 当前存在内容时可打开导出弹窗 |
| 导出方式 | 直接导出 | 将已选内容逐一导出；多选时自动打包下载 |
| 导出方式 | 合并导出 | 将已选内容合并为单张图片导出，视频仅取首帧 |
| 导出格式 | 图片格式 | 支持 PNG、JPG |
| 导出格式 | 视频格式 | 支持 MP4、首帧 PNG、首帧 JPG |
| 导出范围 | 内容选择 | 支持全选/单选/取消选择 |
| 导出执行 | 单个导出 | 单个内容直接下载为对应文件 |
| 导出执行 | 批量导出 | 多个内容自动打包为 ZIP |

## 四、菜单权限
| 权限项 | 说明 | 控制方式 |
|---|---|---|
| 查看导出按钮 | 允许用户打开导出弹窗 | 顶部栏按钮可见 |
| 选择导出范围 | 允许勾选导出内容 | 弹窗内范围面板 |
| 选择导出方式 | 允许切换直接导出/合并导出 | 弹窗内方式下拉 |
| 执行导出 | 允许发起下载 | 底部主按钮 |

## 五、模块需求

### 5.1 导出入口
- 顶部栏「导出」按钮用于打开导出弹窗。
- 当画布中没有任何可见内容时，点击导出应提示“画布中还没有任何内容，请先添加一下内容再导出吧！”。
- 默认导出范围取当前选中内容；若未选中，则默认带入全部可导出内容。

### 5.2 导出方式 [1]
#### 5.2.1 直接导出
- 默认导出方式为「直接导出」。
- 该方式下，选中内容按单个资源逐一导出。
- 当导出范围仅包含 1 个内容时，直接下载单文件。
- 当导出范围包含多个内容时，系统自动将多个导出结果打包为 ZIP 后下载。
- 直接导出时，导出格式区需根据当前已选内容类型动态展示。
- 当已选内容仅包含图片、文本、形状、画板、组合或生图结果等非视频内容时，仅展示图片格式，支持 PNG/JPG。
- 当已选内容同时包含非视频内容与视频内容时，同时展示图片格式与视频格式。图片支持 PNG/JPG；视频支持 MP4、首帧 PNG、首帧 JPG。
- 当已选内容仅包含视频内容时，仅展示视频格式，支持 MP4、首帧 PNG、首帧 JPG。
- 直接导出时，视频若选择“首帧”格式，则导出为图片文件，不再输出视频原文件。

#### 5.2.2 合并导出
- 该方式下，选中内容合成为单张图片。
- 格式仅支持 PNG、JPG，且支持倍率选择：原尺寸、0.5 倍、1.5 倍、2 倍、2.5 倍、3 倍。
- 当选中内容中包含视频时，合并导出应取视频当前首帧参与合成。

### 5.3 导出范围 [3]
- 默认展示已选内容数量，如“已选内容 x/y”。
- 支持勾选全部内容。
- 列表展示每项内容的名称、尺寸、预览缩略图。
- 点击“完成”关闭范围面板，并保留当前勾选状态。

### 5.4 下载结果命名 [4]
- 合并导出文件名使用项目名，后缀为 `.png` 或 `.jpg`。
- 直接导出单文件使用内容名称作为文件名。
- 直接导出多文件时，ZIP 文件名使用项目名，后缀为 `.zip`。
- 文件名中非法字符需替换为安全字符，避免下载失败。

## 六、交互说明
| 场景 | 交互 | 结果 |
|---|---|---|
| 打开弹窗 | 点击顶部「导出」按钮 | 弹出导出作品弹窗 |
| 关闭弹窗 | 点击右上角关闭按钮或点击弹窗外区域 | 关闭弹窗 |
| 切换方式 | 下拉选择「直接导出」或「合并导出」 | 刷新对应导出配置区 |
| 切换格式 [2] | 直接导出下切换图片/视频格式 | 更新对应格式选项与说明 |
| 选择范围 | 勾选单项、全选或取消全选 | 实时更新已选数量 |
| 提交导出 | 点击主按钮 | 触发下载流程 |

## 七、异常与边界
| 异常/边界 | 处理方式 |
|---|---|
| 画布无可见内容 | 阻止打开导出弹窗并提示补充内容 |
| 导出范围为空 | 主按钮置灰，不允许提交 |
| 导出项已隐藏 | 不纳入导出范围 |
| 视频导出为首帧 | 按图片格式输出，不保留视频原文件 |
| 多选直接导出 | 自动打包 ZIP，避免多次弹窗下载 |
| 文件名包含非法字符 | 自动替换后再下载 |

## 八、埋点与日志
本次导出需求优先沿用已有下载埋点事件，不新增新的主事件名。基于 `infinite-ai-canvas` 现有埋点能力，继续使用：
- `design_download_popup_ok`：用户在下载弹窗内点击下载按钮时上报。
- `design_download_popup_done`：下载流程返回结果时上报。

本次新增需求主要补充这两条事件的参数维度，用于表达“直接导出 / 合并导出”以及导出格式区的动态展示情况。

### 8.1 沿用事件与新增参数
| 事件名 | 触发时机 | 新增参数 | 参数说明 | 用途 |
|---|---|---|---|---|
| design_download_popup_ok | 用户点击导出弹窗底部下载主按钮时 | `export_mode` | 导出方式，建议取值：`direct` / `merged` | 区分直接导出与合并导出的选择偏好 |
| design_download_popup_ok | 用户点击导出弹窗底部下载主按钮时 | `format_display_type` | 导出格式区展示形态，建议取值：`image_only` / `image_and_video` / `video_only` | 还原用户点击下载时看到的格式配置界面 |
| design_download_popup_ok | 用户点击导出弹窗底部下载主按钮时 | `selected_content_type` | 已选内容类型，建议取值：`image_like_only` / `image_like_and_video` / `video_only` | 区分不同内容组合下的下载行为 |
| design_download_popup_ok | 用户点击导出弹窗底部下载主按钮时 | `selected_count` | 当前实际选中的导出内容数量 | 分析用户导出范围规模 |
| design_download_popup_ok | 用户点击导出弹窗底部下载主按钮时 | `available_count` | 当前可导出内容总数 | 对比默认选中与实际提交差异 |
| design_download_popup_ok | 用户点击导出弹窗底部下载主按钮时 | `image_export_type` | 图片导出格式，建议取值：`png` / `jpg` / `none` | 统计图片格式使用分布 |
| design_download_popup_ok | 用户点击导出弹窗底部下载主按钮时 | `video_export_type` | 视频导出格式，建议取值：`mp4` / `frame_png` / `frame_jpg` / `none` | 统计视频原文件与首帧导出的使用分布 |
| design_download_popup_ok | 用户点击导出弹窗底部下载主按钮时 | `export_package_type` | 最终导出产物形态，建议取值：`single_file` / `zip` / `merged_image` | 区分单文件、打包下载与合并导出 |
| design_download_popup_ok | 用户点击导出弹窗底部下载主按钮时 | `has_video`、`has_image_like` | 是否包含视频、是否包含非视频内容，建议取值：`是` / `否` | 辅助排查格式展示逻辑与提交配置是否一致 |
| design_download_popup_done | 下载流程返回结果时 | 复用 `design_download_popup_ok` 的全部新增参数 | 保持点击时配置与下载结果可关联 | 支撑导出配置与成功率联合分析 |
| design_download_popup_done | 下载流程返回结果时 | `download_result`、`download_duration` | 沿用现有结果与耗时字段 | 统计不同导出方式与格式的成功率和耗时 |

### 8.2 参数映射规则
| 参数名 | 页面规则 |
|---|---|
| `export_mode` | `source` 映射为 `direct`，`snapshot` 映射为 `merged` |
| `format_display_type` | 当 `showImage=true` 且 `showVideo=false` 时取 `image_only`；当 `showImage=true` 且 `showVideo=true` 时取 `image_and_video`；当 `showImage=false` 且 `showVideo=true` 时取 `video_only` |
| `selected_content_type` | 当已选内容仅包含非视频内容时取 `image_like_only`；当同时包含非视频与视频时取 `image_like_and_video`；当仅包含视频时取 `video_only` |
| `image_export_type` | 仅在展示并选择图片格式时上报 `png` 或 `jpg`，否则上报 `none` |
| `video_export_type` | 仅在展示并选择视频格式时上报 `mp4`、`frame_png` 或 `frame_jpg`，否则上报 `none` |
| `export_package_type` | 直接导出且 `selected_count=1` 时取 `single_file`；直接导出且 `selected_count>1` 时取 `zip`；合并导出时取 `merged_image` |

### 8.3 可选补充项
- 若后续需要分析“打开弹窗但未下载”的漏斗，可再新增 `design_download_popup_show` 曝光事件。
- 本次需求不是必须新增该事件，优先补齐 `design_download_popup_ok` 和 `design_download_popup_done` 的参数即可。

## 九、验收标准
1. 点击顶部「导出」可正常打开导出弹窗。
2. 导出方式中可看到「直接导出」且默认可选。
3. 直接导出下，图片与视频可按各自格式导出。
4. 直接导出下，导出格式区会根据已选内容动态展示为仅图片、图片+视频或仅视频。
5. 直接导出多选时会生成 ZIP，而不是逐个弹出下载。
6. 合并导出下，视频内容按首帧参与合成。
7. 导出范围支持全选、单选、取消选择，且数量展示准确。
8. 埋点可覆盖弹窗打开、模式切换、格式切换、范围调整、提交下载和阻塞场景。
9. 无内容时导出入口有明确提示且不会进入导出流程。

## 十、待确认项
- 当前导出能力是画布级通用能力，后续若要拆分成独立页面，需要再补充页面路由与权限边界。
- 若后续补充“导出记录”“导出失败重试”等能力，本 PRD 需扩展为完整导出管理模块。
- `design_download_popup_done` 的成功口径需进一步确认，是以前端触发浏览器下载为准，还是以后端实际生成成功为准。
