# 公网部署指南

## 目标

将当前项目发布为一个外部可访问的演示地址，并尽量避免依赖 VPN。

当前项目是 Vite 静态前端，最简单的公网发布方式是：

- 腾讯云 EdgeOne Pages / Makers
- Vercel
- Netlify

如果重点是“国内访问尽量稳定”，优先使用腾讯云 EdgeOne。

## 当前项目的部署特征

- 构建命令：`npm run build`
- 构建产物目录：`dist`
- 运行时类型：静态前端

相关文件：

- [package.json](/Users/zhaowenwen/Cursor/创客贴-无限画布/package.json)
- [vite.config.ts](/Users/zhaowenwen/Cursor/创客贴-无限画布/vite.config.ts)

## 先区分两种发布方式

### 1. 公网演示版

适合：

- 演示 UI 与完整流程
- 不要求真实调用 Gemini 生图
- 希望快速拿到一个公网地址

建议：

- 直接部署当前前端
- 不在前端放真实 `GEMINI_API_KEY`
- 使用当前项目里的 mock 生成流程完成演示

### 2. 公网正式版

适合：

- 真正给外部用户使用
- 需要真实生图
- 需要控制密钥、配额、审计和稳定性

必须做的改造：

- 不要在前端直接调用 Gemini
- 将 [src/services/geminiService.ts](/Users/zhaowenwen/Cursor/创客贴-无限画布/src/services/geminiService.ts) 的能力迁移到后端或 Serverless API
- 前端只调用你自己的接口

否则只要前端包对外发布，密钥就存在泄露风险。

## 方案 A：部署到腾讯云 EdgeOne

适用场景：

- 希望国内访问更稳定
- 希望后续可绑定自定义域名
- 希望尽量避免海外链路不稳定

配置：

- Build Command: `npm run build`
- Output Directory: `dist`

建议步骤：

1. 将代码推送到 Git 仓库。
2. 在 EdgeOne Pages / Makers 中导入仓库。
3. 配置构建命令为 `npm run build`。
4. 配置输出目录为 `dist`。
5. 先使用平台生成的预览地址验证。
6. 再绑定自定义域名。

注意：

- 如果你要中国大陆稳定访问并绑定正式域名，通常要考虑备案。
- 如果只是平台分配的临时预览域名，通常可以更快上线做演示。

## 方案 B：部署到 Vercel

适用场景：

- 你想最快拿到一个可公开访问的演示地址
- 主要是给内部或少量外部用户演示

本仓库已补充：

- [vercel.json](/Users/zhaowenwen/Cursor/创客贴-无限画布/vercel.json)

配置：

- Framework Preset: `Vite`
- Build Command: `npm run build`
- Output Directory: `dist`

部署步骤：

1. 将代码推送到 GitHub。
2. 在 Vercel 导入仓库。
3. 选择 Vite 或自动识别。
4. 保持 `npm run build` 与 `dist`。
5. 部署。

## 环境变量建议

如果你只是做演示：

- 不配置真实 `GEMINI_API_KEY`
- 使用本地或演示环境的 mock 流程
- 当前仓库已在 [.env.production](/Users/zhaowenwen/Cursor/创客贴-无限画布/.env.production) 中默认开启 `SKIP_GENERATION=true`

如果你要显式开启跳过真实生成：

- 可以在环境中设置 `SKIP_GENERATION=true`

参考文件：

- [.env.example](/Users/zhaowenwen/Cursor/创客贴-无限画布/.env.example)
- [src/services/generationWorkflow.ts](/Users/zhaowenwen/Cursor/创客贴-无限画布/src/services/generationWorkflow.ts)

## 我建议的实际路线

第一步：

- 先发一个“公网演示版”
- 只验证页面、交互、品牌组和生成后续流程

第二步：

- 将 Gemini 调用迁移到后端
- 再发布“公网正式版”

## 最小部署参数清单

- Install Command: `npm install`
- Build Command: `npm run build`
- Output Directory: `dist`
- Node Version: `20` 或平台默认兼容版本

## 上线前检查

- 确认没有在前端注入真实 `GEMINI_API_KEY`
- 确认 mock 流程可正常完成
- 确认刷新路由不会 404
- 确认图片、静态资源与品牌组展示正常
