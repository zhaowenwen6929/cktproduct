# PRD 文档规范

本文档用于统一需求文档的写法、粒度和验收口径，确保产品、设计、开发、测试可以基于同一份内容协作。

## 1. 编写原则

- 先写结论，再写细节。
- 只写可执行、可验证、可回溯的内容。
- 不写模糊表述，避免使用“尽量”“差不多”“优化一下”等词。
- 需求必须能回答三个问题：做什么、为什么做、怎么验收。
- 一份 PRD 只描述一个业务目标或一组强关联需求。

## 2. 标准结构

PRD 建议按以下顺序编写：

1. 文档目的
2. 适用范围
3. 背景与目标
4. 术语定义
5. 整体流程
6. 功能范围
7. 业务规则
8. 页面交互与状态说明
9. 异常与边界处理
10. 数据与接口说明
11. 埋点与日志
12. 验收标准
13. 测试用例
14. 待确认事项

## 3. 各章节写法

### 3.1 文档目的

说明这份 PRD 要解决什么问题，面向谁使用。

要求：

- 1 到 3 句话说明背景。
- 写清楚文档用途，例如产品对齐、开发联调、测试验收。

### 3.2 适用范围

说明本次需求覆盖哪些页面、模块、端、角色。

要求：

- 写清楚包含范围。
- 同时写清楚不包含范围，避免需求外溢。

### 3.3 背景与目标

说明为什么要做这件事，以及期望达成什么结果。

要求：

- 背景写现状问题。
- 目标写结果导向，不写实现手段。
- 目标最好可衡量。

### 3.4 术语定义

统一关键名词的含义，避免多人理解不一致。

要求：

- 对状态、对象、角色、业务名词进行定义。
- 同一词汇在全文保持一致。

### 3.5 整体流程

用流程图或步骤描述核心链路。

要求：

- 主流程必须完整。
- 关键分支必须补齐。
- 有状态流转的需求，必须标明状态迁移顺序。

### 3.6 功能范围

描述系统要做哪些事，每个功能的触发条件、输入、输出和约束是什么。

要求：

- 按模块拆分。
- 每个模块单独描述。
- 触发条件、行为、结果要成对出现。

### 3.7 业务规则

描述与金额、积分、权限、排序、状态、计算相关的规则。

要求：

- 用明确条件表达，例如“当 A 且 B 时，执行 C”。
- 写清默认值、优先级、覆盖关系、例外情况。

### 3.8 页面交互与状态说明

描述页面上的按钮、弹窗、提示、状态展示、禁用逻辑。

要求：

- 按组件或区域写，不要按散句写。
- 写清点击前、点击后、加载中、失败、空态、成功态。
- 若有多状态流转，必须列出状态机。

### 3.9 异常与边界处理

描述输入异常、网络异常、权限异常、并发异常、超时等情况。

要求：

- 每种异常都要有提示文案或处理策略。
- 明确是重试、兜底、拦截还是直接失败。

### 3.10 数据与接口说明

描述涉及的字段、接口、请求参数、返回值和依赖关系。

要求：

- 字段名、类型、是否必填、默认值要写清楚。
- 若接口未定稿，要明确标记“待后端确认”。

### 3.11 埋点与日志

描述需要采集的行为事件和关键日志。

要求：

- 事件名、触发时机、参数、用途要写清楚。
- 埋点要和验收目标对齐。

### 3.12 验收标准

描述需求是否完成的判定条件。

要求：

- 用可检查的句子写，不写主观感受。
- 每条标准尽量能被测试直接验证。

### 3.13 测试用例

列出主流程、分支流程、异常流程的测试点。

要求：

- 至少覆盖正常流程、边界条件、异常场景。
- 测试点要对应前文需求，不要额外扩展。

### 3.14 待确认事项

列出当前还未明确、需要产品/研发/设计补充确认的内容。

要求：

- 只写真实未决事项。
- 每条都要有负责人或确认方向。

## 4. 粒度规范

- 一个 PRD 只解决一个明确目标。
- 一个模块内的需求要聚合描述，避免拆成零散段落。
- 一条规则只表达一个判断条件。
- 一条流程只描述一个完整链路。

## 5. 文案规范

- 状态文案要短，且和实际行为一致。
- 错误文案要告诉用户发生了什么、下一步怎么做。
- 按钮文案要和行为一致，避免“确认”“提交”“继续”等泛化词混用。
- 同一业务概念在全文只用一个名称。

## 6. 图示规范

- 流程复杂时建议补流程图。
- 状态较多时建议补状态机图。
- 页面结构复杂时建议补页面结构图或交互图。
- 图只是辅助，文字必须能独立说明需求。

## 7. 验收检查清单

发布前建议检查以下内容：

- 是否写清楚需求背景和目标。
- 是否写清楚适用范围和不包含范围。
- 是否补齐主流程和关键分支。
- 是否补齐业务规则和异常处理。
- 是否补齐验收标准和测试点。
- 是否存在模糊词、口语化表达或实现导向描述。
- 是否可以仅靠本文档完成开发和测试理解。
